# Crystallization

Crystallization is the process of compressing a domain into multiple
resolutions so an agent can load the right amount of context for the task.

## Layers

- Layer 4: `seed` for domain DNA and heuristics
- Layer 3: `principles` for rules, frameworks, and decision trees
- Layer 2: `knowledge` for curated guides and references
- Layer 1: `sources` for provenance and deeper reading
- Layer 0: `raw` for unprocessed material

## Why This Helps

- lower context cost for routine tasks
- clearer trust boundaries than naive retrieval
- faster bootstrapping into domain awareness
- better separation between public framework and private expertise

## Output Files

- `seed.md`
- `principles.md`
- `graph.yaml`
- `persona.md`

These files can be created with local templates first, then refined by humans
or model-assisted workflows.
